# Findings

## When Meta-Prompting Helps

* Converts vague requirements into structured prompts.
* Saves time when onboarding new projects.
* Ensures consistency across different use cases.
* Helps create reusable prompt templates quickly.
* Useful when client requirements are incomplete or ambiguous.

## When Manual Prompting Is Faster

* Requirements are already detailed and well defined.
* Small one-off tasks.
* Highly specialized workflows requiring domain expertise.
* Situations where precise control over prompt wording is required.

## Observations

* The generated prompts generally contained all six required sections.
* The prompts were detailed and production-ready.
* Very vague briefs can cause the model to make assumptions about the use case.
* Prompt quality improved when the brief was more specific.

## Conclusion

Meta-prompting is an effective technique for automatically generating structured prompts from vague project requirements. It improves consistency and reduces prompt engineering effort, while manual prompting remains valuable for highly customized or specialized tasks.
